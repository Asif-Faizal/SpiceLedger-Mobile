# Architecture

SpiceLedger Mobile follows **feature-first Clean Architecture** with three layers per feature and shared `core/` infrastructure.

## Layer diagram

```
┌─────────────────────────────────────────────────────────────┐
│                    presentation                             │
│  Pages, Widgets, Bloc/Cubit (UI state + user events)        │
└──────────────────────────┬──────────────────────────────────┘
                           │ calls
┌──────────────────────────▼──────────────────────────────────┐
│                      domain                                 │
│  Entities, Repository interfaces, Use cases                 │
└──────────────────────────┬──────────────────────────────────┘
                           │ implemented by
┌──────────────────────────▼──────────────────────────────────┐
│                       data                                  │
│  Repository impl, Remote data sources, Models (DTOs)         │
└──────────────────────────┬──────────────────────────────────┘
                           │
                    Dio / GraphQLClient
                           │
                    SpiceLedger Backend
```

## Feature modules

| Feature | Path | Responsibility |
|---------|------|----------------|
| **onboarding** | `lib/features/onboarding/` | Splash animation, first-run onboarding, route to auth or home |
| **auth** | `lib/features/auth/` | Email check, login, register, profile, logout |
| **admin** | `lib/features/admin/` | Admin shell, dashboard metrics, product/grade/price management |
| **merchant** | `lib/features/merchant/` | Merchant shell, profile, merchant details CRUD |

Each feature follows:

```
feature/
├── data/
│   ├── datasources/       # HTTP / GraphQL I/O
│   ├── models/            # JSON models (+ .g.dart, .freezed.dart)
│   └── repositories/      # *RepositoryImpl
├── domain/
│   ├── entities/          # Pure business objects
│   ├── repositories/      # Abstract contracts
│   └── usecases/          # Single-purpose actions
└── presentation/
    ├── bloc/ or cubit/    # State management
    ├── pages/             # Screens
    └── widgets/           # Feature-specific UI
```

## Core package (`lib/core/`)

| Module | Files | Role |
|--------|-------|------|
| **config** | `env_config.dart` | `ApiConfig`, `EnvironmentConfig` from `--dart-define` |
| **di** | `injection.dart`, `injection.config.dart` | `getIt` service locator; generated by `injectable` |
| **network** | `network_module.dart`, `graphql_module.dart`, `token_interceptor.dart`, `graphql_error_link.dart`, `error_handler.dart`, `api_response.dart` | HTTP clients, auth refresh, error mapping |
| **storage** | `secure_storage.dart`, `hive_module.dart` | JWT storage; Hive `spice_cache` box |
| **theme** | `app_theme.dart`, `components/` | Colors, typography, buttons, snackbars |
| **error** | `failures.dart` | Typed `Failure` hierarchy for `Either` |
| **widgets** | `shimmer_widget.dart`, `searchable_dropdown.dart` | Reusable UI |

## State management

| Pattern | Used for |
|---------|----------|
| **Bloc** | Multi-step flows with events (`LoginBloc`, `AdminProductsBloc`, `AdminDashboardBloc`, `MerchantDetailsBloc`, `ProductActionBloc`) |
| **Cubit** | Simpler state (`SplashCubit`, `CheckEmailCubit`, form cubits, navigation cubits) |
| **freezed** | Immutable states/events (`admin_products_state`, `dashboard_model`, etc.) |

Repositories return `Either<Failure, T>` (`dartz`). Presentation maps failures to error UI / snackbars.

## Dependency injection

Registration is code-generated from annotations:

- `@injectable` / `@lazySingleton` / `@Injectable(as: Interface)` on classes
- `@module` on `NetworkModule`, `GraphQLModule`, `StorageModule`

Bootstrap in `main.dart`:

```dart
await configureDependencies();  // getIt.init()
```

Resolve in UI: `getIt<SomeBloc>()` or `getIt<SomeUseCase>()`.

## Navigation

No declarative router. Flow is driven by:

1. **`SplashCubit`** — reads secure storage + Hive → emits auth role or onboarding
2. **`SplashScreen` listener** — `pushReplacement` to `AdminBucketView`, `MerchantBucketView`, `CheckEmailPage`, or `OnboardingPage`
3. **Bucket views** — `IndexedStack` + bottom nav (`AdminBucketView`, `MerchantBucketView`)
4. **Auth pages** — linear `Navigator.push` (check email → login/register)

Global `navigatorKey` in `main.dart` is used by `TokenInterceptor` to redirect to splash on failed token refresh.

## Data models vs entities

| Layer | Type | Example |
|-------|------|---------|
| **Model** (data) | JSON-serializable, `fromJson` / `toJson` | `UserModel`, `ProductModel`, `MerchantModel` |
| **Entity** (domain) | Immutable, no framework deps | `UserEntity`, `ProductEntity`, `MerchantEntity` |

Models expose `.toEntity()` at the repository boundary.

## Storage keys (secure)

Written by `AuthRepositoryImpl` on login:

| Key | Content |
|-----|---------|
| `access_token` | JWT access token |
| `refresh_token` | JWT refresh token |
| `user_id` | Account ID |
| `user_type` | `admin` or `merchant` (enum name) |
| `name` | Display name |

Hive `spice_cache` box:

| Key | Content |
|-----|---------|
| `has_seen_onboarding` | `bool` — skip onboarding after first run |
